  Session Behavior
                                                                                                                                                                       
  - The session begins in the current working directory with access to shell history
  - Shell state (environment variables, aliases, etc.) does not persist between commands                                                                               
  - The cwd persists between commands but not across sessions                                                                                                          
  - Long-running commands can use run_in_background: true

  - Default to no comments — well-named identifiers do the work                                                                                                        
  - Prefer early returns and guard clauses
  - No error handling for impossible states                                                                                                                            
  - Trust internal code and framework guarantees
  - Only validate at system boundaries (user input, external APIs)                                                                                                     
                  
  Tool Philosophy                                                                                                                                                      
                  
  - Read before editing, Grep before asking                                                                                                                            
  - Dedicated tools (Glob, Grep, Read) are better than shell commands (ls, cat, find, grep) for their domain
  - Edit does exact string replacement and fails if the string isn't unique                                                                                            
  - Write overwrites existing files — read first, then overwrite
